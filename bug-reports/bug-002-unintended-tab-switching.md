Unintended tab switching while scrolling between "Live Channels" and "Categories"

Bug Type: UX / Usability

Severity: Medium

Priority: Medium

Steps to reproduce:
1. Tap on the Search icon
2. Search for any tag or key word
3. Select "Live Channels" tab
4. Scroll down at a decent speed standar speed for an User

Expected result:
- User should be able to stay in the "Live Channels" tab with out switching to any other tab while vertically scrolling
- Switching tabs should require an explicit horizontal swipe or intentional gesture

Actual result:
- While scrolling vertically, the application unintentionally switches to the Categories tab

Environment:
- App: Twitch
- OS: Android
- App Version: 27.5.0
